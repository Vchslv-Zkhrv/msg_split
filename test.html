But when you set formatter as html5, certain characters will be encoded as HTML entities, so the output might be different to the input. Your original start and end offsets could be off. If your html isn't perfectly-formed, the parser will try to fix it, so the output won't be the same as the input. See the documentation - Differences between parsers. For these 2 reasons, you should parse the original html, output it, and then parse that output. That way the input should always be the same as the output.
